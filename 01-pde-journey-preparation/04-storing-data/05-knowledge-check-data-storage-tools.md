# Knowledge Check: Storing the Data

## Serverless Analytics with SQL
**Mr. X:** Cymbal Retail collects a lot of data and wants to store and analyze it in a **serverless** and **cost-effective** way. Analysts also want to use **SQL**. Which tool fits best?  
**Mr. Artificial King:** When you hear *serverless analytics* and *SQL at scale*, one tool stands out. **BigQuery** is a fully managed, serverless data warehouse that lets analysts run powerful SQL queries without managing any infrastructure.

**✔ Correct Answer:** BigQuery

## Centralized Storage for All Data Types
**Mr. X:** Cymbal Retail also has structured, semi-structured, and unstructured data. They want one central, low-cost place to store everything. What should they use?  
**Mr. Artificial King:** For storing **all types of data**—files, images, logs, videos, and raw datasets—the best option is **Cloud Storage**. It’s highly scalable, durable, and cost-effective, making it ideal as a centralized data repository or data lake.

**✔ Correct Answer:** Cloud Storage

## Key Takeaways
**Mr. Artificial King:** Remember:
- Use **BigQuery** for serverless analytics with SQL  
- Use **Cloud Storage** as a centralized, cost-effective storage layer for any data type  

Choosing the right storage tool based on access patterns and data types is a core skill for a Professional Data Engineer.
